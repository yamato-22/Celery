# Домашнее задание к лекции «Celery»

Специалист по машинному обучению создал модель для апскейлинга
изображений и функцию, которую позволяет ее использовать.  
`EDSR_2.pb` - модель.  
В модуле `upscale.py` находится функция `upscale`, которая имплементирует модель.  
В файле `requirements.txt` перечислены зависимости

Задача написать сервис для апскейлинга изображений на базе Flask, Celery и ИИ модели.
Должно быть реализовано 3 роута
- POST `/upscale`. Принимает файл с изображением и возвращает id задачи
- GET `/tasks/<task_id>` возвращает статус задачи и ссылку на обработанный файл, если задача выполнена
- GET `/processed/{file}` возвращает обработанный файл

# Задание выполнено (с учетом замечаний проверяющего)

- Использовал io.BytesIO для передачи содержимого файла в OpenCV без сохранения на диск
- Также обработанный файл не сохраняется на диск, содержимое в бинарном виде возвращается из 
GET запроса после окончания выполнения task.

1. Реализовано 2 роута. 
- POST `/upscale`. Принимает файл с изображением и возвращает id задачи celery
- GET `/tasks/<task_id>` возвращает текущий статус задачи и обработанный файл 
в бинарном виде, если задача выполнена
- *GET `/processed/{file}` роут удален из проекта, т.к. лишен смысла,* 
*обработанный файл возвращается в бинарном виде из Get запроса.*

2. Приложение докеризовано.
3. Состав приложения: папка APP c кодом, файлы для докеризации, 
файл .env.example - образец для создания env файла, 
request_example.py - файл демонстрирующий работу приложения,
